FlightRadar GraphQL API

FlightRadar GraphQL API Documentation

This API provides real-time and historical aircraft position data for flight tracking and analysis.

Basic Usage

To query data, use the root Query object. You can:

  • Fetch the latest aircraft positions with a limit
  • Filter aircraft data by ICAO code (aircraft identifier)
  • Filter aircraft data by flight number (callsign)

Example Query

query {
  aircraftPositions(limit: 5) {
    flight
    latitude
    longitude
    altitude
    timestamp
  }
}

Queries

aircraftByFlight

Description

Returns aircraft position history by flight identifier (e.g., callsign or flight number).

Example: { aircraftByFlight(flight: "SAS123", limit: 5) { latitude longitude timestamp } }

Response

Returns [AircraftPositions]

Arguments
Name Description
flight - String!
limit - Int!

Example

Query
query AircraftByFlight(
  $flight: String!,
  $limit: Int!
) {
  aircraftByFlight(
    flight: $flight,
    limit: $limit
  ) {
    id
    icao
    flight
    latitude
    longitude
    altitude
    speed
    heading
    timestamp
  }
}
Variables
{"flight": "xyz789", "limit": 987}
Response
{
  "data": {
    "aircraftByFlight": [
      {
        "id": 4,
        "icao": "xyz789",
        "flight": "abc123",
        "latitude": 123.45,
        "longitude": 123.45,
        "altitude": 123.45,
        "speed": 987.65,
        "heading": 987.65,
        "timestamp": "abc123"
      }
    ]
  }
}

Types

AircraftPositions

Description

Represents the positional and flight data of an aircraft at a specific point in time.

Fields
Field Name Description
id - ID! Internal incremental identifier for this position record (e.g., 1, 2, 3...). Primarily used for internal database reference.
icao - String ICAO 24-bit address of the aircraft (hexadecimal string).
flight - String Flight number or callsign of the aircraft.
latitude - Float Current latitude of the aircraft in decimal degrees.
longitude - Float Current longitude of the aircraft in decimal degrees.
altitude - Float Altitude of the aircraft in feet.
speed - Float Ground speed of the aircraft in knots.
heading - Float Heading or track angle of the aircraft in degrees.
timestamp - String Timestamp of the data point in ISO 8601 format (e.g., "2025-06-26T12:34:56Z").
Example
{
  "id": 4,
  "icao": "xyz789",
  "flight": "xyz789",
  "latitude": 987.65,
  "longitude": 123.45,
  "altitude": 987.65,
  "speed": 987.65,
  "heading": 123.45,
  "timestamp": "abc123"
}

Types

ID

Description

The ID scalar type represents a unique identifier, often used to refetch an object or as key for a cache. The ID type appears in a JSON response as a String; however, it is not intended to be human-readable. When expected as an input type, any string (such as "4") or integer (such as 4) input value will be accepted as an ID.

Example
"4"
Types

Int

Description

The Int scalar type represents non-fractional signed whole numeric values. Int can represent values between -(2^31) and 2^31 - 1.

Example
987
Types

String

Description

The String scalar type represents textual data, represented as UTF-8 character sequences. The String type is most often used by GraphQL to represent free-form human-readable text.

Example
